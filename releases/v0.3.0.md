## What's new in v0.3.0

This release adds a dedicated Cache stats tab, moves the stats dashboard onto precomputed SQLite rows for faster rendering, and fixes token accounting across both Claude and Codex providers. Activity tracking is now timezone-aware, prompt growth charts focus on main-thread turns, and Codex token semantics are normalized to match Anthropic's exclusive-bucket convention — eliminating double-counting that made cross-provider comparisons misleading.

### Features

- **Cache stats tab** — A new tab between Tools and Performance surfaces prompt-cache behavior. Summary chips show cache read/write totals, hit rate, miss rate, and reuse ratio. Four chart lanes break down daily hit rate and reuse trends (toggled with `m`), main-vs-subagent hit rate comparison, cache reuse by session duration, and hit rate decay over session length. Codex sessions derive cache write tokens from OpenAI's auto-cache semantics when the API omits explicit write counts.

  ![Cache stats tab showing daily hit rate, main-vs-subagent comparison, and duration histograms](https://raw.githubusercontent.com/rkuska/carn/v0.3.0/releases/assets/v0.3.0/cache.png)

- **Precomputed stats rows** — The stats UI now reads performance sequences, turn metrics, and activity data from canonical SQLite rows instead of loading and parsing transcript blobs on every tab switch. A one-time backfill runs automatically on first launch after upgrade.

- **Message timestamps** — Per-message timestamps are now preserved through the Claude and Codex projection paths and stored in canonical transcript blobs, enabling time-attributed token analysis in the activity and stats views.

### Fixes

- **Codex token double-counting** — OpenAI reports input_tokens as total (cached + uncached) and output_tokens as total (reasoning + non-reasoning), while Anthropic reports each bucket exclusively. The Codex parser now normalizes at parse time so TotalTokens, cache miss cost, and overview totals are correct without provider-specific branches downstream.

- **Sidechain model metadata leak** — Primary Claude sessions no longer inherit model identifiers from sidechain assistant records. Empty models no longer leave stray separators in browser metadata rendering.

- **UTC minute activity buckets** — Activity tracking switched from precomputed daily rows to UTC minute buckets, enabling correct timezone rebucketing after load. Streaks no longer extend beyond the selected time range due to out-of-range activity.

- **Prompt growth aligned to main-thread turns** — Only non-empty main-thread user prompts advance the turn position. Subagents, sidechains, system records, and tool-result-only steps are excluded so the chart reflects actual user interaction depth.

### Improvements

- **Stats hot path overhead reduced** — Overview keeps a fixed-size top-five during scan instead of sorting all candidates, cache aggregation uses indexed day buckets, and turn token growth preallocates per-session slices. Stats box rendering uses direct border drawing instead of lipgloss layout.

- **Degraded stats notification** — When precomputed row queries fail, the stats view now shows an error notification with rebuild guidance instead of silently dropping derived views. Successful recompute clears the warning automatically.

- **Full timestamps in carn.log** — Log output uses RFC3339 format with date, time, and numeric timezone offset instead of bare clock times.
